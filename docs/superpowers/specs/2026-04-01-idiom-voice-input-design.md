# 成语猜猜猜语音输入功能设计（离线）

## 1. 目标与范围

在当前 `idiom_game.py` 中新增**离线语音输入**能力，首版仅覆盖**单人模式中“输入完整成语”**这一输入点。

目标：
- 支持本地离线识别（不依赖云服务）
- 不破坏现有键盘输入流程
- 识别不可用时自动降级为键盘输入

不在本次范围：
- 对战模式语音输入
- 创新模式语音输入
- 语音唤醒词与持续后台监听

## 2. 方案选型

采用方案：**实时监听线程 + 按住空格说话，松开提交**

理由：
- 用户交互直观，符合你确认的操作方式；
- 主线程逻辑改动小，语音模块可独立封装；
- 能保持键盘输入为主通道，降低故障风险。

## 3. 架构设计

新增模块（单文件内分区实现）：

1. `VoiceEngine`（能力探测与初始化）
   - 检查 `vosk`、`sounddevice`、模型目录是否可用
   - 对不可用场景返回明确原因（缺依赖/缺模型/无麦克风）

2. `VoiceListener`（实时监听线程）
   - 负责麦克风流读取与 Vosk 识别
   - 使用线程安全队列缓存最新识别文本
   - 支持 `start_listening()` / `stop_listening()` / `latest_text()`

3. `voice_input_once()`（一次语音采集流程）
   - 引导“按住空格说话，松开提交”
   - 在监听期间更新候选识别结果
   - 松开后提交识别文本；若为空则回退键盘输入

4. `safe_guess_input()`（接入点）
   - 单人模式猜词改为通过该函数获取答案
   - 默认仍为键盘输入，用户可选语音

## 4. 交互流程（单人模式）

在“请输入完整成语”阶段：

1. 显示输入方式：
   - `T` 键盘输入（默认）
   - `V` 语音输入
2. 选择 `V` 后进入语音界面：
   - 提示“按住空格说话，松开提交”
   - 实时监听并保留最近识别结果
3. 提交逻辑：
   - 若识别到文本：返回该文本参与判题
   - 若未识别到：提示后自动切回键盘输入

## 5. 依赖与配置

依赖包：
- `vosk`
- `sounddevice`

模型目录（建议）：
- `models\vosk-model-small-cn\`

配置常量（新增）：
- `VOICE_ENABLED_DEFAULT = True`
- `VOICE_MODEL_PATH = "models\\vosk-model-small-cn"`
- `VOICE_SAMPLE_RATE = 16000`

## 6. 错误处理与降级

以下场景自动降级到键盘输入并提示：
- 未安装 `vosk` / `sounddevice`
- 语音模型路径不存在
- 麦克风不可用或打开失败
- 识别超时/无结果

原则：
- 语音失败不影响主流程；
- 不吞异常，给出可理解提示；
- 游戏可继续完整运行。

## 7. 测试设计

单元测试（可 mock）：
- 语音能力检测函数在不同依赖缺失场景返回正确状态
- 语音返回空文本时正确回退键盘
- `safe_guess_input()` 在 T/V 分支行为正确

交互烟测：
- 单人模式下选择 V，按住空格说话后可提交文本
- 禁用/缺失模型时提示并自动回退键盘
- 现有扩展功能不回归（计分、学习、限时、排行榜）

## 8. 风险与规避

风险：
- 不同 Windows 终端对“按住空格”键盘事件支持差异
- 本地麦克风权限导致采集失败

规避：
- 预留回退交互：若检测到按键监听不可用，自动改为“按一次开始，再按一次结束”
- 明确打印故障原因，避免用户误判为游戏卡死

## 9. 验收标准

- 单人模式可用语音提交答案（离线）
- 语音不可用时可平滑回退到键盘输入
- 原有基础/扩展/加分功能不受破坏
