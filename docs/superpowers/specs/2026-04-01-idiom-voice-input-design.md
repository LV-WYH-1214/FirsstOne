# 成语游戏语音输入设计（按住空格识别）

## 1. 问题与目标

当前游戏仅支持键盘输入。目标是在不破坏现有玩法与测试稳定性的前提下，新增“按住空格开始语音识别、松开空格结束并提交文本”的输入方式，并在识别失败时自动回退键盘输入。

## 2. 范围

### In Scope

- 仅新增“语音转文本输入”能力（替代文本输入）。
- 触发方式：在输入阶段按住空格进行识别，松开结束。
- 失败策略：识别为空或不合法时，提示并回退键盘输入。
- 采用离线识别：Vosk 本地模型。
- 保持现有单人、双人、创新模式业务流程不变。

### Out of Scope

- 语音播报题目/提示（TTS）。
- 全语音菜单导航。
- 在线语音服务或联网依赖。

## 3. 设计方案（已选）

推荐并确认方案：**事件监听空格按下/抬起**（体验最佳）。

- 空格按下：开始采集麦克风音频并送入识别器。
- 空格抬起：停止采集，输出本次识别文本。
- 若文本为空或不符合当前输入规则：提示原因，切回原有键盘输入。

## 4. 架构与组件

在 `idiom_game.py` 内新增输入适配层，不改业务主流程。

### 4.1 `VoiceInputEngine`

职责：封装离线语音识别细节。

- 初始化与能力检查：Vosk、麦克风、模型路径。
- 生命周期：`start_listen()` / `stop_listen_and_transcribe()`。
- 对外仅暴露字符串结果与明确错误信息，不吞异常。

### 4.2 `InputController`

职责：在“当前输入点”统一路由语音与键盘。

- 默认使用键盘输入。
- 检测到空格按住事件后，委托 `VoiceInputEngine` 获取一次识别文本。
- 调用现有校验函数（`parse_int_in_range`、`normalize_single_char_input` 等）进行合法性判断。
- 不合法或异常时给出提示并立即回退键盘流程。

## 5. 与现有代码的集成点

- 保持 `choose_mode`、`play_round`、`play_battle_mode`、`play_description_challenge_mode` 等业务函数语义不变。
- 在 `safe_input`、`safe_single_char_input`、`safe_int_input` 读取路径中引入统一输入控制器，减少调用点改动。
- 原有输入校验逻辑复用，不新增重复规则。

## 6. 错误处理与降级策略

以下情况均**不终止游戏**，直接提示并回退键盘输入：

- Vosk 未安装
- 模型目录缺失/无效
- 麦克风设备不可用
- 识别结果为空
- 识别结果不满足当前输入约束（如双人模式必须单字符）

提示遵循现有项目风格：明确原因 + 下一步动作（“请改为键盘输入”）。

## 7. 依赖与配置

- Python 依赖：`vosk`、用于麦克风采集与按键监听的库（如 `sounddevice`、`keyboard`）。
- 模型约定：默认从固定本地目录加载（例如 `material/vosk-model`），若不存在则提示并降级。
- 不引入网络请求，不收集用户隐私数据到外部服务。

## 8. 测试策略

### 单元测试

- 新增输入路由测试：空格触发语音分支、失败回退键盘分支。
- 使用 mock/stub 替代真实麦克风与识别器，避免环境依赖。
- 覆盖关键失败路径：空结果、非法结果、初始化失败。

### 回归测试

- 运行现有 `tests/test_idiom_game.py`，确认既有行为不变。
- 新增测试不修改现有断言语义，仅补充语音适配层验证。

## 9. 成功标准

- 用户可在输入阶段按住空格说话，松开后得到可用于当前题型的文本。
- 识别失败场景可稳定回退键盘输入，无崩溃、无死循环。
- 原有功能与测试保持通过。
