# 成语猜猜猜全功能设计说明

## 1. 背景与目标
当前项目已完成基础玩法和扩展功能 1/2/3/4/5/7。目标是在不破坏现有体验的前提下，补齐：

- 扩展功能 6：排行榜系统（前五名，文件持久化）
- 扩展功能 8：对战模式（双人）

并最终达到“基础功能 + 扩展 1-8 全覆盖”。

## 2. 设计范围
本次只覆盖代码功能，不包含实验报告材料（函数关系图、代码优缺点分析）。

包含内容：

- 单人模式保留并兼容已有扩展 1/2/3/4/5/7
- 新增双人对战模式并接入主入口
- 新增排行榜读写、排序、前五展示
- 每局结束后自动显示前五排行榜

不包含内容：

- 多文件重构或面向对象大改
- GUI 或网络联机
- 超出题目要求的额外玩法

## 3. 总体架构
采用“适度重构（单文件分层）”方案，在 `idiom_game.py` 内拆分职责但不改项目结构：

- 模式入口层：选择单人模式或双人对战模式
- 单人游戏层：沿用并封装现有流程（难度/限时/学习/分类/计分）
- 对战游戏层：双人回合制猜字并判胜负
- 排行榜层：统一处理记录、读取、排序、写回、展示
- 公共工具层：输入校验、题库加载、文本掩码等

这样可以最小化回归风险，同时提升可维护性。

## 4. 功能设计

### 4.1 单人模式（保持现有能力）
单人模式继续支持：

- 基础功能：随机题目、释义提示、3 次机会、猜错展示答案
- 扩展 1：难度隐藏 1/2/3 字
- 扩展 2：计分（猜对 +10、提示 -2、连胜 3 局 +5）
- 扩展 3：分类选择
- 扩展 4：30 秒限时
- 扩展 5：学习模式错题解释与例句
- 扩展 7：从 `idioms.txt` 读取

### 4.2 排行榜系统（新增扩展 6）
新增文件：`leaderboard.txt`  
记录格式：`姓名:分数`（每行一条）

行为规则：

- 每局结束后，将本局参与玩家写入排行榜
- 读取历史记录并合并后按分数降序排序
- 仅保留前五名并回写文件
- 立即打印当前前五名

容错规则：

- 文件不存在：按空榜处理并在首次写入时创建
- 脏数据行：跳过并给出提示，不影响正常运行
- 分数解析失败：该行忽略

### 4.3 双人对战模式（新增扩展 8）
流程：

1. 输入玩家 1、玩家 2 姓名
2. 随机抽取同一成语作为目标词
3. 初始化显示为全下划线（按字位）
4. 两位玩家轮流操作，每次猜“位置 + 汉字”
5. 猜对则揭示该位置；猜错则轮到对方
6. 首个将目标词完整揭示者获胜

回合规则：

- 同一位置已揭示时，要求玩家重新输入
- 非法位置或空输入循环重输
- 可设置总回合上限防止极端僵局；超限则按“揭示字数更多者胜”，再平局则判平

积分与榜单：

- 对战胜者记一条分数（例如 +10）进入排行榜
- 对战结束后立即刷新并展示前五

## 5. 数据与文件

- `idioms.txt`：继续支持 `成语:描述:分类[:例句]`
- `score.txt`：保留单人历史最高分
- `leaderboard.txt`：新增多玩家前五榜单

文件读写均采用 UTF-8，并保持显式错误提示。

## 6. 错误处理策略

- 输入异常：通过统一输入函数反复校验，不允许空值和非法选项直通
- 文件异常：不存在/坏行/坏分数均显式提示并继续可运行
- 模式分流异常：未知模式值回退到重新选择
- 运行稳定性：不使用吞异常的宽泛兜底，保持可见错误信息

## 7. 测试与验收
采用交互回归方式逐项验收：

- 基础功能：正确触发与输赢路径
- 扩展 1-5、7：在单人模式逐项验证
- 扩展 6：排行榜写入、排序、截断前五、脏行容错
- 扩展 8：双人轮流猜字、胜负判定、榜单联动

验收标准：

- 作业要求的基础 + 扩展 1-8 均可独立触发并行为正确
- 新增功能不破坏已实现功能
- 文件持久化结果与显示一致
